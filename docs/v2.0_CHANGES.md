# TG AI Poster v2.0 - Comprehensive Refactor Documentation

## Summary

This document describes the comprehensive architectural refactor of the content generation system, transforming it into a professional AI-powered editorial system with maximum content quality.

---

## Phase 1: Pipeline Architecture Refactor

### New Pipeline Stages

The pipeline has been refactored into the following sequential stages:

```
news_fetch → source_verification → context_builder → llm_draft_generation → editor_review → media_prompt_generation → quality_checker → telegram_formatter → analytics → publisher
```

### Stage Descriptions

| Stage | File | Description |
|-------|------|-------------|
| **news_fetch** | `pipeline/source_collector.py` | Collects raw articles from RSS feeds, APIs, and aggregators |
| **source_verification** | `pipeline/source_verification.py` | Scores sources on trust tiers (1-3),
 validates URLs, requires min 2 sources |
| **context_builder** | `pipeline/context_builder.py` | **NEW** - Assembles structured context object with verified facts, entities, and confidence scores |
| **llm_draft_generation** | `pipeline/llm_generator.py` | Generates draft with **GLM-5** (via `claude-cli` provider), temperature 0.15 (factual) or 0.4 (analysis),
 strict JSON response |
| **editor_review** | `pipeline/editor_review.py` | Second LLM pass to fix issues, add original observation |
| **media_prompt_generation** | `pipeline/media_prompt_generator.py` | **NEW** - Generates cinematic image prompts with specific exclusions |
| **quality_checker** | `pipeline/quality_rules.py` | **NEW** - Runs all 50 quality rules |
| **telegram_formatter** | `pipeline/telegram_formatter.py` | **NEW** - Converts JSON to Telegram-ready format with block validation |
| **analytics** | `pipeline/analytics.py` | **NEW** - Tracks engagement, generates reports, manages post type rotation |
| **publisher** | `publisher/bot_publisher.py` | Publishes with retry and audit logging |

## Phase 2: Master LLM System Prompt

### LLM Configuration

**Model:** GLM-5 (via Claude Code CLI with Coding Plan)
**Provider:** `claude-cli`
**Base URL:** Claude Code CLI interface

**Temperature Settings:**
- `breaking` / `tool_roundup`: 0.15 (factual, precise)
- `deep_dive` / `analysis`: 0.4 (more creative, analytical)

### File: `llm/prompts/post_generator.txt`

The new system prompt enforces:
- Russian language output
- Original reconstruction (no copying)
- Technology explanation focus
- Source traceability
- Confidence score marking for low-confidence facts
- Cliche avoidance
- Short paragraphs (2-4 sentences)
- Concrete subject in title
- Industry trend connection in analysis
- Self-contained TLDR

### Key JSON Schema Fields

```json
{
  "title": "max 120 chars with 1-2 emojis",
  "hook": "1-2 sentences answering what + why care",
  "body": "800-1500 chars, short paragraphs",
  "key_facts": ["exactly 4 standalone verifiable claims"],
  "analysis": "2-4 sentences with industry trend + question",
  "sources": [{"name", "url", "confidence"}],
  "useful_links": [{"label", "url"}],
  "media_prompt": "detailed cinematic image prompt",
  "tldr": "single sentence, self-contained summary",
  "hashtags": ["company", "technology", "AI"],
  "needs_review": boolean,
  "post_type": "breaking|deep_dive|tool_roundup|analysis"
}
```
